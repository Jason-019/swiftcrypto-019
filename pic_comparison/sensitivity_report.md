## 1. 测试方法论

### 测试流程

```
origin_1.jpg
    │
    ├─ 施加调整 (亮度/对比度/饱和度/...)
    │
    ▼
调整后图像
    │
    ├─ 1. 参考行检测 (滑动窗口 + 自适应背景阈值)
    ├─ 2. 参考色采样 (从8个参考块中心加权取色)
    ├─ 3. RGB → CIE Lab 转换
    ├─ 4. a*b* 欧氏距离逐像素匹配
    ├─ 5. 逐行解码 → b64 字符串
    └─ 6. 校验和验证 + 裁剪重试
    │
    ▼
   PASS / FAIL
```

### 解码成功标准

- 参考行成功检测（找到 8 个等间距亮色块）
- 参考色成功采样（8 个色块均有足够像素）
- 解码出完整 b64 字符串
- 校验和通过（含最多 3 字符裁剪重试）

### 注意事项

- 各维度独立测试，不叠加
- 每项从"绝对破坏"到"原始值"再到"极端增强"逐步扫描
- 报告中的"通过范围"表示该区间内**所有**测试点均通过

## 2. 亮度敏感性 (Brightness)

### 定义

**亮度调整**：对图像所有像素的 RGB 三通道乘以相同系数 `factor`。

```
output = pixel × factor
```

- `factor = 1.0`：原始图像
- `factor = 0.5`：亮度减半（变暗）
- `factor = 2.0`：亮度加倍（变亮）

### 物理含义

模拟拍摄环境光照变化、显示器亮度调节、曝光补偿等。亮度变化主要影响 CIE Lab 中的 **L\* 通道**（明度），对 a\*b\*（色度）影响较小——这正是算法使用 a\*b\* 欧氏距离匹配的原因。

| **0.40×** | ✅ | **暗端边界** |
| 0.50× | ✅ | |

### 通过范围: **0.40× ~ 2.50×** *(容差比 6.25:1)*

### 分析

- **暗端失效 (0.30×)**：参考色块亮度低于自适应背景阈值 (`bgAvg+40`)，检测算法无法在背景中分离参考块
- **亮端失效 (3.00×)**：高亮通道裁切至 255，多个颜色的 R/G 通道同时饱和，失去区分度。例如黄色 (255,214,10) 在 3.00× 后变为 (255,255,30)，与橙色 (255,149,0)→(255,255,0) 几乎相同

## 3. 对比度敏感性 (Contrast)

### 定义

**对比度调整**：以中灰 (128,128,128) 为中心，拉伸或压缩像素值。

```
output = (pixel - 128) × factor + 128
```

- `factor = 1.0`：原始对比度
- `factor = 0.5`：对比度降低 50%（画面变灰）
- `factor = 2.0`：对比度翻倍（鲜艳更艳，暗部更暗）

### 物理含义

模拟雾霾、逆光拍摄、图像后期处理中的对比度滑块。对比度改变会同时影响 L\* 和 a\*b\*——这是算法相对薄弱的维度，因为 a\*b\* 的绝对值被压缩，但**方向（色相）保持不变**。

### 测试结果

| 倍率 | 结果 | 说明 |
|------|------|------|
| 0.10× | ❌ NO_REF | 参考块与背景几乎同色 |
| 0.20× | ❌ NO_REF | |
| 0.30× | ❌ NO_REF | |
| 0.40× | ❌ NO_REF | |
| 0.50× | ❌ NO_REF | |
| 0.60× | ❌ NO_REF | |
| **0.70×** | ✅ | **暗端边界** |
| 0.80× | ✅ | |
| 0.90× | ✅ | |
| 1.00× | ✅ | 原始图像 |
| 1.10× | ✅ | |
| 1.20× | ✅ | |
| 1.30× | ✅ | |
| 1.40× | ✅ | |
| 1.50× | ✅ | |
| 1.60× | ✅ | |
| 1.80× | ✅ | |
| **2.00×** | ✅ | **亮端边界** |

### 通过范围: **0.70× ~ 2.00×** *(容差比 2.86:1)*

### 分析

- **对比度降低失效 (≤0.60×)**：背景像素从中灰附近向 128 靠拢，背景与参考块的亮度差缩小。当背景亮度超过自适应阈值时，检测算法无法分离。**这不是颜色匹配失败，而是参考行检测失败。** 滑动窗口检测在此前版本中已修复了"卡片边框被误认"的问题，但极低对比度下背景本身变成了"亮区"。
- **对比度增强无上限 (2.00× 仍通过)**：增强对比度使颜色更鲜明，a\*b\* 距离更大，匹配更容易。理论上可以更高，测试范围未覆盖。

## 4. 饱和度敏感性 (Saturation)

### 定义

**饱和度调整**：保留亮度信息，仅改变色彩鲜艳程度。

```
gray = (R + G + B) / 3
output = gray + (pixel - gray) × factor
```

- `factor = 1.0`：原始饱和度
- `factor = 0.0`：完全灰度（黑白）
- `factor = 0.05`：仅保留 5% 色彩

### 物理含义

模拟老旧照片褪色、相机饱和度设置、图像去色处理。**CIE Lab 空间的核心优势**：饱和度变化主要影响 a\*b\* 的幅值（magnitude），不改变 a\*b\* 的方向（色相角）。a\*b\* 欧氏距离匹配在幅值缩放时仍能正确分类。

### 测试结果

| 倍率 | 结果 | 说明 |
|------|------|------|
| 0.00× | ❌ NO_REF | 完全灰度，参考块无颜色 |
| **0.05×** | ✅ | **仅 5% 饱和度即可解码！** |
| 0.10× | ✅ | |
| 0.15× | ✅ | |
| 0.20× | ✅ | |
| 0.25× | ✅ | |
| 0.30× | ✅ | |
| 0.40× | ✅ | |
| 0.50× | ✅ | |
| 0.60× | ✅ | |
| 0.70× | ✅ | |
| 0.80× | ✅ | |
| 0.90× | ✅ | |
| 1.00× | ✅ | 原始图像 |

### 通过范围: **5% ~ 100%** *（仅 5% 色彩即可恢复）*

### 分析

这是算法最突出的优势。即使 95% 的色彩褪去，a\*b\* 向量方向不变，欧氏距离匹配依然能正确分类。0.00× 失败是因为灰度图没有色度信息，参考块无法与背景区分——这是物理极限，任何颜色编码都无法避免。

## 5. 色温敏感性 (Color Temperature)

### 定义

**色温调整**：改变蓝-黄轴的颜色平衡，模拟光源色温变化。

```
R_new = R + shift
B_new = B - shift
G 保持不变
```

- `shift = 0`：原始
- `shift = +40`：暖色（红增蓝减，模拟低色温/钨丝灯）
- `shift = -40`：冷色（蓝增红减，模拟高色温/阴天）

### 物理含义

模拟不同光源（日光/白炽灯/荧光灯）、手机相机的白平衡偏移、滤镜效果。色温变化主要影响 a\*b\* 空间中的 b\* 轴（蓝-黄）。

### 测试结果

| 偏移 | 结果 |
|------|------|
| -80 | ✅ OK |
| -60 | ✅ OK |
| -40 | ✅ OK |
| -30 | ✅ OK |
| -20 | ✅ OK |
| -10 | ✅ OK |
| 0 | ✅ OK |
| +10 | ✅ OK |
| +20 | ✅ OK |
| +30 | ✅ OK |
| +40 | ✅ OK |
| +60 | ✅ OK |
| +80 | ✅ OK |

### 通过范围: **-80 ~ +80** *(全覆盖)*

### 分析

色温偏移在所有测试范围内均通过。原因：

1. **参考色采样自适应**：解码器从实际参考块中采样颜色，而非使用硬编码 RGB。色温偏移后，参考块和数据格被同等影响，采样色自动跟踪变化。
2. **a\*b\* 空间覆盖广**：8 种颜色在 a\*b\* 平面上分布范围大（从 -60 到 +76），±80 的偏移不足以使颜色合并。

## 6. 噪声敏感性 (Gaussian Noise)

### 定义

**高斯噪声**：对每个像素的每个通道添加独立的高斯分布随机值。

```
output = pixel + N(0, sigma^2)
```

- `sigma = 0`：无噪声
- `sigma = 5`：轻微噪点（肉眼几乎不可见）
- `sigma = 25`：明显噪点
- `sigma = 60`：严重噪点（画面粗颗粒）

### 物理含义

模拟高 ISO 拍摄、传感器热噪声、多次转发压缩积累的噪点。

### 测试结果

| sigma | 结果 |
|-------|------|
| 0 | ✅ OK |
| 5 | ✅ OK |
| 10 | ✅ OK |
| 15 | ✅ OK |
| 20 | ✅ OK |
| 25 | ✅ OK |
| 30 | ✅ OK |
| 40 | ✅ OK |
| 50 | ✅ OK |
| **60** | ✅ OK |

### 通过范围: **σ = 0 ~ 60** *(全覆盖)*

### 分析

噪声容忍度意外地高，原因有三：

1. **中心加权采样**：每个 cell 采样 5px 半径（~121 像素），中心像素权重高。随机噪声在大量像素中趋于抵消。
2. **a\*b\* 欧氏距离**：噪声在 RGB 空间均匀分布，但在 Lab 空间中主要影响 L\*（亮度噪点），对 a\*b\* 影响较小。
3. **校验和裁剪重试**：即使少量 cell 误读，尾部裁剪机制可纠正。

## 7. 模糊敏感性 (Gaussian Blur)

### 定义

**高斯模糊**：对图像进行高斯核卷积，模拟失焦或运动模糊。

```
output = GaussianBlur(pixel, radius)
```

- `radius = 0`：无模糊
- `radius = 1.0px`：轻微模糊
- `radius = 3.0px`：严重模糊

### 物理含义

模拟对焦不准、手抖、图像缩放放大。

### 测试结果

| 半径 | 结果 |
|------|------|
| 0 | ✅ OK |
| 0.3px | ✅ OK |
| 0.5px | ✅ OK |
| 0.7px | ✅ OK |
| **1.0px** | ✅ OK |
| 1.2px | ❌ MISMATCH |
| 1.5px | ❌ MISMATCH |
| 2.0px | ❌ MISMATCH |
| 2.5px | ❌ MISMATCH |
| 3.0px | ❌ MISMATCH |

### 通过范围: **0 ~ 1.0px**

### 分析

模糊超过 1.0px 时，20×20px 的 cell 边界颜色混合严重——相邻 cell 的不同颜色开始渗透。中心加权采样（半径 5px）能抵抗轻微模糊，但 cell 之间的 3px 间隙不足以为强模糊提供隔离。

## 8. 暗角敏感性 (Vignette)

### 定义

**暗角**：图像边缘逐渐变暗，中心保持原始亮度。

```
dist = 像素到中心的归一化距离 (0~1)
vignette = 1.0 - strength × dist
output = pixel × vignette
```

- `strength = 0`：无暗角
- `strength = 0.5`：边缘变暗 50%
- `strength = 0.9`：边缘几乎全黑

### 物理含义

模拟广角镜头暗角、闪光灯覆盖不均、Instagram 风格的暗角滤镜。

### 测试结果

| 强度 | 结果 |
|------|------|
| 0 | ✅ OK |
| 0.1 | ✅ OK |
| 0.2 | ✅ OK |
| 0.3 | ✅ OK |
| 0.4 | ✅ OK |
| 0.5 | ✅ OK |
| 0.6 | ✅ OK |
| 0.7 | ✅ OK |
| 0.8 | ✅ OK |
| **0.9** | ✅ OK |

### 通过范围: **0 ~ 90%** *(全覆盖)*

### 分析

暗角容忍度极高的关键：**自适应背景阈值**。参考色块位于图像顶部（y≈39），不在最暗的边缘区域。自适应阈值从参考行附近采样背景亮度，而非整张图的平均。即使边缘极暗，只要参考行区域的背景与色块仍有区分度，检测就不会失败。这在 `modified_2.jpg`（真实暗角滤镜图片）上已得到验证。

## 9. JPEG 压缩敏感性

### 定义

**JPEG 压缩**：以指定质量参数重新编码图像，模拟多次保存或社交平台压缩。

```
output = JPEGEncode(JPEGDecode(image), quality)
```

- `quality = 100`：最高质量（几乎无损）
- `quality = 50`：中等质量
- `quality = 10`：极低质量（严重块效应和色彩量化）

### 物理含义

模拟微信/QQ/微博等社交平台的图片压缩，多次转发中的累积质量损失。

### 测试结果

| 质量 | 结果 |
|------|------|
| **10** | ✅ OK |
| 15 | ✅ OK |
| 20 | ✅ OK |
| 25 | ✅ OK |
| 30 | ✅ OK |
| 40 | ✅ OK |
| 50 | ✅ OK |
| 60 | ✅ OK |
| 70 | ✅ OK |
| 80 | ✅ OK |
| 90 | ✅ OK |
| 100 | ✅ OK |

### 通过范围: **Q=10 ~ 100** *(全覆盖)*

### 分析

JPEG 压缩的核心操作是颜色子采样和 DCT 系数量化。8 色编码的固有大色块（20×20px）天然抗 DCT 块效应——单个 cell 覆盖多个 JPEG 宏块（16×16px），中心加权采样使 DCT 边界伪影被边缘像素低权重自然衰减。色彩量化对 8 种精心挑选的高饱和度颜色影响极小。

### 通过范围一览

|------|----------|--------|------|
| JPEG 质量 | Q10 ~ Q100 | 10:1 | 🟢 极强 |

| 暗角 | 0 ~ 90% | — | 🟢 极强 |
| 色温 | -80 ~ +80 | — | 🟢 极强 |
| 噪声 | σ=0 ~ 60 | — | 🟢 极强 |
| 亮度 | 0.40× ~ 2.50× | 6.25:1 | 🟢 强 |
| 对比度 | 0.70× ~ 2.00× | 2.86:1 | 🟢 强 |
| 模糊 | 0 ~ 1.0px | — | 🟡 中等 |

### 算法架构对抗性的贡献

| 组件 | 主要对抗 | 贡献 |
|------|----------|------|
| CIE Lab a\*b\* 匹配 | 亮度变化 | 核心：L\* 与 a\*b\* 分离，亮度不影响色度匹配 |
| 参考色实际采样 | 色温/偏色 | 关键：自适应跟踪实际颜色，无需硬编码 |
| 自适应背景阈值 | 暗角/亮度 | 本地化背景采样，不依赖全局亮度 |
| 滑动窗口检测 | 对比度降低 | 修复：背景变亮时排除边框伪亮区 |
| 中心加权采样 | 噪声/模糊 | 抑制边缘像素，强化 cell 中心信号 |
| maxDataRows 限制 | 所有退化 | 防止 footer 区域被误读为数据 |
| 校验和裁剪重试 | 尾部误读 | 最后防线：1~3 个尾部垃圾字符自动纠正 |

### 已知局限

1. **对比度 < 0.70×**：背景与参考块亮度差消失，检测端（非匹配端）失效。根本原因是 8 色编码使用亮度阈值来分离色块和背景——当背景本身变亮时，阈值区分能力下降。

2. **模糊 > 1.0px**：cell 间 3px 间隙不足以抵御强模糊。如需更高抗模糊能力，可增大 cell 间隙或 cell 尺寸（但会降低数据密度）。
