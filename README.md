# 🐈 SwiftCrypto — Taylor Swift 主题加密聊天工具

> *"I don't trust nobody and nobody trusts me"* — Look What You Made Me Do

**SwiftCrypto** 是一款以 Taylor Swift 为主题的端到端加密通信 PWA。将密码学变成一场 Eras 巡演——用 13 套专辑色卡加密消息、把密文藏进 MIDI 旋律中伪装成钢琴曲、用 167 道 Swiftie 冷知识生成密码。**全部运算在浏览器本地完成，零服务器，零追踪。**

---

## ✨ 功能亮点

### 🔐 加密 / 解密

| 特性 | 说明 |
|------|------|
| **AES-256-GCM** | 工业级对称加密，每次加密生成 12 字节随机 IV，相同明文每次产生不同密文 |
| **PBKDF2 密钥派生** | 60 万次 SHA-256 迭代 + 固定盐值，将任意长度密码派生为 256 位密钥 |
| **ECDH 双重加密** 🛡️ | 可选高安全模式：双方各生成 P-256 密钥对，交换身份码后密钥 = 密码原文 + ECDH 共享密钥。即使密码意外泄露，没有对方身份码也无法解密 |
| **中文密码** 🇨🇳 | 完整支持中文/日文/Emoji 密码，IME 输入法友好，实时显示字符数 + UTF-8 字节数 |
| **定时焚毁** 🔥 | 消息附带自毁时间戳（10 秒 ~ 71 时 59 分 59 秒），到期后自动显示「已焚毁」 |
| **Emoji 密文** 🎭 | Base64 密文 → 64 种 Emoji 表情，外观像普通聊天，支持自动识别反向转换 |

### 📤 多种分享方式

#### 🎨 Eras 色块名片 — 抗干扰图像编码

> 专为微信/QQ 等有损压缩 IM 平台设计，经过 7 维度敏感性分析验证

Eras 码将密文编码为**8 色矩形色块矩阵**，每个 20×20px 色块代表一个 Base64 字符。**不依赖 QR 码库**——这是从零实现的自主颜色编码方案。

**编码流程：**
```
密文 → Base64 字符 → 64 色索引 → 8 色块矩阵（每 3 bit 一种颜色）
       ↓
参考行（顶部 8 个等间距亮色块，用于解码时颜色采样）
       ↓
数据行（每行 16 个 cell，cell 间距 3px，行间距 2px）
       ↓
校验边框（6 位十六进制颜色码，用作定位标识 + 校验和）
```

**解码算法核心：**
```
参考行检测 → 8 参考色块采样 → RGB → CIE Lab 转换
                                    ↓
每个数据 cell 中心加权采样（半径 5px，约 121 像素）
                                    ↓
a*b* 欧氏距离匹配 → 最近参考色 → Base64 字符
                                    ↓
逐行拼接 → 校验和验证 → 尾部裁剪重试（最多 3 字符）→ 输出密文
```

**为什么用 CIE Lab 而非 RGB？**
- CIE Lab 将亮度（L\*）与色度（a\*b\*）分离
- 亮度变化、曝光差异只影响 L\*，不干扰色度匹配
- a\*b\* 欧氏距离与人眼感知色差一致

**经过 7 维度敏感性分析验证（详见 `pic_comparison/sensitivity_report.md`）：**

| 维度 | 通过范围 | 评级 |
|------|----------|------|
| 🗜 **JPEG 压缩** | Q=10 ~ 100 | 🟢 极强 — 微信最差质量也能过 |
| 🎨 **饱和度** | 仅需 5% | 🟢 极强 — 95% 褪色仍可解码 |
| 🌓 **暗角** | 0 ~ 90% | 🟢 极强 — Instagram 滤镜随便加 |
| 🌡 **色温** | -80 ~ +80 | 🟢 极强 — 白炽灯/阴天全覆盖 |
| 📺 **高斯噪声** | σ=0 ~ 60 | 🟢 极强 — 严重噪点不碍事 |
| ☀️ **亮度** | 0.40× ~ 2.50× | 🟢 强 — 6 倍亮度范围 |
| 🔍 **模糊** | 0 ~ 1.0px | 🟡 中等 — 轻微失焦可过 |

**关键设计决策：**
- **参考色实时采样**而非硬编码 RGB——色温/白平衡偏移后，参考块和数据格被同等影响，采样自动跟踪
- **自适应背景阈值**——从参考行附近本地采样背景亮度，不依赖全局亮度，暗角 90% 仍可检测
- **中心加权采样**（半径 5px）——cell 中心像素权重高，边缘像素自然衰减，抵抗噪声和轻微模糊
- **3px cell 间隙**——在数据密度和抗模糊之间取平衡
- **校验和裁剪重试**——尾部 1~3 个误读字符自动纠正

**使用流程：** 加密 → 生成 Eras 名片 → 截图 → 发微信/QQ → 接收方保存 → 文件导入 → 自动识别（先尝试 QR 解析，失败回退 Eras 解码）

#### 📱 二维码图片

- 280×280px QR 码，**M 级纠错**（15% 容错，约 1160 字符容量），M 级撑不下自动降级 L 级
- 中央 🔐 图标：跟随主题色的渐变圆角方块 + 白色 🔐 文字，品牌辨识度
- **密码提示**：可附加最多 60 个中文字符提示（Base64 嵌入 JSON 载荷），扫码后自动显示
- **容量显示**：按钮上实时显示「N/1160 字」，超长时橙色警告
- 支持 Web Share API 分享 PNG 或保存到相册
- 通过 html2canvas（2x 分辨率）截图保证清晰度

#### 📝 纯文字 / 🔗 链接

- 复制 Base64 密文或生成带密文参数的 URL
- 附加明文密码提示
- Eras 链接附带预渲染色块 Canvas

#### 🎵 MIDI 隐写 — 密文藏进旋律

> 最隐蔽的分享方式：外观是普通的钢琴曲 .mid 文件，听觉上与原曲毫无差别

**编码原理：**
将密文 Base64 字符映射为 MIDI Note On 事件的 **velocity（力度）** 值（0~63，对应 64 个 Base64 字符）。音符的时值和音高保持原曲不变——**听起来和原始 MIDI 完全一样**，只有解析 velocity 值才能提取密文。

**容量：** 一首 3 分钟钢琴曲约 500~2000 个音符，可编码约 500~2000 个字符。

**瀑布图可视化：** Canvas 实时绘制音符瀑布图（音高 Y 轴、时间 X 轴），蓝色线条显示被编码密文的音符分布，直观掌握容量。

**预缓存系统 📦：**
- 一键「📦 预缓存 MIDI」将所有歌曲下载到 Cache Storage
- 缓存后**完全离线可用**——无网络也能编码、解码、试听
- 状态实时显示「已缓存 N 首 / 共 N 首」
- 智能增量下载，已有缓存自动跳过
- 适配 PWA 离线策略，Service Worker 优先从缓存响应

**分享流程：** 编码 → 生成 .mid → 通过微信/邮件/AirDrop 发送 → 接收方文件导入 → 输入密码 → 解密 → 可试听收到的 .mid

## 🛡️ 安全架构

| 层级 | 技术 | 说明 |
|------|------|------|
| **密钥派生** | PBKDF2-SHA256 | 60 万次迭代 + 固定盐，任意密码 → 256 位密钥 |
| **对称加密** | AES-256-GCM | 随机 12 字节 IV，GCM 认证标签防篡改 |
| **非对称增强** | ECDH P-256 | 双重加密：密码 + ECDH 共享密钥混合派生 |
| **身份码** | JWK P-256 公钥 | Base64 → 二维码交换 → 导入对方公钥 |
| **密码强度** | 熵值评估 | 分析字符集大小 × 长度，惩罚常见弱密码/键盘行走/顺序字符/Leet-speak，输出 bit + 6 级评级 |
| **存储** | localStorage / Cache API | 密码历史、聊天记录、HS 密钥、MIDI 缓存；隐身模式全部跳过 |

### 隐私保证
- ✅ 密码**永不离开浏览器**——加解密全部在 Web Crypto API 中完成
- ✅ 纯静态 PWA，无任何后端请求，完全离线可用
- ✅ 隐身模式 👻：关闭后不留密码历史、聊天记录
- ✅ 草稿模式 📝：可选保留输入框内容，刷新不丢
- ✅ 完全开源，可审计
