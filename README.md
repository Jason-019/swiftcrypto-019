# 🐈 SwiftCrypto — Taylor Swift 主题加密通信网站

> *"You knew the password, so I let you in the door."* — hoax, folklore

**SwiftCrypto** 是一款以 Taylor Swift 为主题的端到端加密通信渐进式 Web 应用。将密码学变成一场 Eras 巡演——用 13 套专辑色卡加密消息、把密文藏进 MIDI 钢琴曲、用 167 道 Swiftie 冷知识生成密码、用歌词作共享密钥。**全部运算在浏览器本地完成，零服务器，零追踪。**

---

## ✨ 功能总览

### 🔐 加密 / 解密

| 特性                         | 说明                                                                  |
| ---------------------------- | --------------------------------------------------------------------- |
| **AES-256-GCM**        | 工业级对称加密，12 字节随机 IV，相同明文每次不同密文                  |
| **PBKDF2 密钥派生**    | 60 万次 SHA-256 迭代 + 固定盐，任意密码 → 256 位密钥                 |
| **ECDH 双重加密** 🛡️ | 可选高安全模式：P-256 ECDH + 密码原文混合密钥。即使密码泄露也无法解密 |
| **中文/Emoji 密码**    | 完整支持中文、日文、Emoji 密码，IME 友好                              |
| **定时焚毁** 🔥        | 消息附带自毁时间（10 秒 ~ 71 时 59 分），到期自动"已焚毁"，倒计时可见 |
| **Emoji 密文** 🎭      | Base64 ↔ 64 种 Emoji 双向映射，外观像普通聊天                        |
| **密码强度评估**       | zxcvbn 熵值分析，6 级评级 + 破解时间估算（8×RTX 5090 GPU）           |

### 📤 多种分享方式（4 种）

#### 🎨 Eras 色块名片 — 抗干扰图像编码

> 专为微信/QQ 有损压缩设计，**从零实现的自主颜色编码方案**，不依赖 QR 码库。

将密文编码为 **8 色矩形色块矩阵**：每个 20×20px 色块代表一个 Base64 字符，顶部参考行用于解码时颜色采样，底部含时间戳哈希和主题签名。

**核心技术：CIE Lab 色彩空间 a\*b\* 欧氏距离匹配**

- L\*（明度）与 a\*b\*（色度）分离——亮度变化不影响颜色匹配
- a\*b\* 欧氏距离与人眼感知色差一致
- 参考色实时采样而非硬编码 RGB——色温/白平衡偏移后自动跟踪

**7 维度敏感性分析验证（详见 `pic_comparison/sensitivity_report.md`）：**

| 维度         | 通过范围        | 评级                            |
| ------------ | --------------- | ------------------------------- |
| 🗜 JPEG 压缩 | Q=10 ~ 100      | 🟢 极强 — 微信最差质量也能过   |
| 🎨 饱和度    | 5% ~ 100%       | 🟢 极强 — 95% 褪色仍可解码     |
| 🌓 暗角      | 0 ~ 90%         | 🟢 极强 |
| 🌡 色温      | -80 ~ +80       | 🟢 极强 —色温滤镜几乎全覆盖    |
| 📺 高斯噪声  | σ=0 ~ 60       | 🟢 极强 — 严重噪点不碍事       |
| ☀️ 亮度    | 0.40× ~ 2.50× | 🟢 强 — 6.25 倍容差            |
| 🔍 模糊      | 0 ~ 1.0px       | 🟡 中等 — 轻微失焦可过         |

**抗干扰关键设计：** 中心加权采样（半径 5px，约121 像素）抵抗噪声和轻微模糊；3px cell 间隙平衡数据密度与抗模糊；自适应背景阈值从参考行附近本地采样，暗角 90% 仍可检测；校验和裁剪重试自动纠正尾部 1-3 个误读字符。

#### 📱 二维码图片

- 280×280px QR 码，M 级纠错（15% 容错），M 级撑不下自动降级 L 级
- 中央 🔐 图标：主题色渐变方块 + 白色 🔐 文字
- 可附加最多 60 字密码提示
- 支持 Web Share API 分享或保存 PNG

#### 📝 纯文字 / 🔗 分享链接

- 生成带密文 + 密码提示的 URL
- 链接可附带歌词提示：专辑名（`&al=`）、歌曲名（`&sg=`）、歌词行索引（`&li=`）、尝试次数限制（`&tr=`）
- 对方打开链接自动弹出解密界面，含歌词自动补全和模糊匹配反馈

#### 🎵 MIDI 隐写 — 密文藏进旋律

> 最隐蔽的分享方式：外观是普通钢琴曲 .mid，听觉与原曲毫无差别。

- Base64 字符映射为 MIDI Note On **velocity（力度）** 值（0~127）
- 音符时值和音高保持原曲不变——**听起来完全一样**
- 3 分钟钢琴曲约 500~2000 字符容量
- Canvas 瀑布图可视化编码音符分布
- 一键预缓存 200+ 首 Taylor Swift 钢琴 MIDI，缓存后完全离线可用

### 🔑 密码生成（4 种方式）

| 方式                 | 说明                                                                   |
| -------------------- | ---------------------------------------------------------------------- |
| 🪶**歌词挑选** | 内置完整歌词库，按专辑 → 歌曲 → 歌词行三级自定义下拉，选一句作密码   |
| 💎**趣味问答** | 167 道 Taylor Swift 冷知识（专辑/歌词/巡演/猫咪/格莱美…），答对即密码 |
| 🎲**随机生成** | 从歌词库中安全随机拼出`word+digits+word` 格式密码                    |
| ✍️**自定义** | 任意文本密码，实时强度评估                                             |

**歌词分享提示系统：** 选择歌词后可勾选 📀 专辑 / 🎵 歌曲 / 🔒 限制尝试次数，自动嵌入分享链接。接收方看到提示但不知道完整密码——靠模糊匹配合格才能解密。ℹ️ 按钮弹出 Olivia 猫管家口吻的详细说明。

**模糊匹配：** 接收方输入不必一字不差——算法计算相似度，接近时提示"差不多了！再试一次？"，完全匹配才解密。结合专辑/歌曲提示，既给方向又不作弊。

### 🛡️ 隐私与安全

| 模式                     | 说明                                                                   |
| ------------------------ | ---------------------------------------------------------------------- |
| 👻**隐身模式**     | 不保存密码历史、聊天记录、草稿。关闭即焚                               |
| 📝**草稿模式**     | 自动保存/恢复密码、明文、密文输入框内容，刷新不丢                      |
| 🛡️**高安全模式** | ECDH P-256 密钥对 + 密码原文混合，需交换身份码。即使密码泄露也无法解密 |

**隐私保证：** 密码永不离开浏览器（Web Crypto API）、纯静态 PWA 无后端、完全开源可审计。

## 🧬 技术栈

| 层          | 技术                                             |
| ----------- | ------------------------------------------------ |
| 加密        | Web Crypto API (AES-256-GCM, PBKDF2, ECDH P-256) |
| QR 生成     | qrcodejs (M/L 纠错自适应)                        |
| QR 扫描     | jsQR (摄像头实时 + 图片文件)                     |
| MIDI 播放   | Tone.js                                          |

| 密码强度    | zxcvbn                                           |
| 架构        | 单文件 PWA (~3500 行)，Service Worker 离线缓存   |
